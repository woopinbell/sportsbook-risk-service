## 작업 내역

### 빌드 정의 (pom.xml)

부모로 `spring-boot-starter-parent`를 상속하지 않고 **BOM을 import**했다. shared-protocol이
라이브러리 pom 스타일로 그렇게 했고, 일관성을 위해 따랐다. C/C++에서 CMake `find_package`로
버전을 외부에 맡기는 것과 비슷하게, BOM이 버전을 관리하므로 개별 의존성에 `<version>`을 거의
안 적는다.

```xml
<dependencyManagement>
  <dependencies>
    <dependency>
      <groupId>org.springframework.boot</groupId>
      <artifactId>spring-boot-dependencies</artifactId>
      <version>${spring-boot.version}</version>
      <type>pom</type>
      <scope>import</scope>
    </dependency>
    <!-- testcontainers BOM도 동일하게 import -->
  </dependencies>
</dependencyManagement>
```

런타임 의존: shared-protocol(value object + Avro 이벤트 POJO), web + validation + actuator,
spring-data-redis(Lettuce), spring-kafka, lombok, micrometer-prometheus, OTel exporter,
logstash-logback-encoder. 테스트: spring-boot-starter-test(JUnit5+AssertJ+Mockito),
spring-kafka-test, testcontainers(kafka). **Redis는 Testcontainers 공식 모듈이 없어
`GenericContainer`로 띄운다** — 그래서 redis 모듈 의존이 따로 없다.

처음에 Avro를 명시 의존으로 적었다가 `'dependencies.dependency.version' for
org.apache.avro:avro is missing` 에러를 받았다. Spring Boot BOM이 Avro 버전을 관리하지 않기
때문. shared-protocol을 통해 transitive로 들어오므로 명시 의존을 **지웠다**.

### 정책 yaml (application.yml)

ADR-0009의 type-safe config 방향. Redis/Kafka 연결과 함께 `risk.limits` / `risk.patterns`
기본값을 둔다. per-currency stake 한도, single-bet max, open-exposure, selections-per-minute,
패턴 3종(enabled + 임계값). 이 값들은 다음 커밋의 `@ConfigurationProperties` record가 바인딩한다.

### 진입점 + Clock 빈

```java
@SpringBootApplication
@ConfigurationPropertiesScan   // policy record를 명시 등록 없이 발견
@EnableKafka
public class RiskServiceApplication {
  public static void main(String[] args) { SpringApplication.run(...); }

  @Bean
  Clock clock() { return Clock.systemUTC(); }  // 테스트에서 fixed clock으로 교체 가능
}
```

`Clock`을 빈으로 노출한 건 sliding window counter와 패턴 룰을 **고정 시계(fixed clock)**로
단위 테스트하기 위해서다. C/C++에서 `time()`을 직접 부르면 테스트가 시간에 의존해 깨지기
쉬운데, 시계를 주입(inject)하면 "2026-05-28 10:00:00 정각"을 강제할 수 있다.

처음엔 `main`만 있는 utility class였는데 checkstyle `HideUtilityClassConstructor`가 막았다.
`Clock` @Bean을 추가하면서 자연히 해소됐다(이제 인스턴스 메서드가 있는 일반 클래스).

## 요약

- BOM import 방식 pom (부모 상속 안 함).
- Avro는 shared-protocol에서 transitive — 명시 의존 금지.
- Redis는 `GenericContainer`로 테스트(공식 모듈 없음).
- `Clock` 빈으로 시간 의존 테스트 가능하게.
- CLAUDE.md는 .gitignore 박제.
