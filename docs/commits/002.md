# 002 · .claude/ 세션 디렉터리 무시

> commit `07c652a` — `chore(project): ignore .claude/ session directory`

## 개요

작은 housekeeping 커밋. Claude Code 세션이 긴 작업 중 `.claude/`에 `scheduled_tasks.lock`과
로컬 `settings.local.json`을 떨군다. CLAUDE.md와 같은 부류의 AI 툴링 산출물이므로 같은
무시 규칙으로 보낸다.

자기 미화 금지 원칙대로 적으면: 이 커밋의 diff는 `.gitignore` 한 줄(`+.claude/`)이 전부다.
커밋을 분리한 이유는 무관한 변경을 섞지 않기 위해서 — 마침 sliding window counter 작업을
시작하려던 참에 working tree에 `.claude/`가 untracked로 떠서, 기능 커밋에 딸려 들어가지
않도록 먼저 단독으로 처리했다.

## 작업 순서

1. working tree에서 `.claude/` 발견 (`git status`).
2. `.gitignore`에 `.claude/` 라인 추가 (CLAUDE.md 바로 밑).
3. 기능 파일이 같이 스테이징되지 않도록 `.gitignore`만 단독 커밋.

## 작업 내역

```gitignore
# AI session context — operational file, not a project artifact
CLAUDE.md
.claude/
```

`git restore --staged`로 counter 작업 파일들을 unstage하고 `.gitignore`만 남겨 커밋했다.
"한 커밋 = 한 가지 변경" 원칙(SESSION_STARTUP.md)을 지키기 위한 분리.

## 결과

`.gitignore` 1줄 추가. `.claude/` 이하 어떤 것도 GitHub에 올라가지 않는다.

## 요약

- AI 툴링 산출물(`.claude/`)을 git에서 제외.
- diff는 한 줄 — 무관한 변경 분리를 위해 단독 커밋.
