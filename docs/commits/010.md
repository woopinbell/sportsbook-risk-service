placeholder("측정 후 박제")를 실제 표로 교체:

```markdown
| 시나리오 | metric | 측정값 | 목표 |
| check @ 1000 RPS | p50/p95/p99 | 2.34 / 11.03 / 30.21 ms | < 5 / < 15 / < 30 |
| check @ 5000 VU  | p95/p99     | 1.46 s / 1.73 s         | 운영 한계 |
| Kafka produce    | throughput  | 173,010 events/s        | 100,000 |
```

p99 0.21ms 미달을 숨기지 않고 미달 사실 + 개선 동선(Lua GC tuning / Lettuce shared vs pool /
warm-up)을 같이 적었다(자기 미화 금지).

`risk_pattern_flags_total{rule,action}`)을 문서화해, 운영자가 Grafana 대시보드(추후
orchestration repo에 박제)에 무엇을 꽂을지 알 수 있게 했다.
