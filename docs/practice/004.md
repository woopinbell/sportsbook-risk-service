# 004 · per-subject 한도 override + 정책 fallback resolver

## 개요

counter(003)가 "지금 얼마 썼나"를 준다면, 이 커밋은 "한도가 얼마인가"를 준다. 운영자(admin-api)가
특정 user/market에 거는 override를 Redis hash에 저장하고, `LimitResolver`가 **override →
yaml 기본값** 순으로 effective limit을 합성한다.

## 작업 순서

1. `LimitOverrideStore` 인터페이스 — 저장 추상화(테스트 더블 교체 가능).
2. `RedisLimitOverrideStore` — Redis hash 구현.
3. `LimitResolver` — override 위에 정책 fallback 합성.
4. Mockito 단위 테스트(resolver) + Testcontainers 테스트(store).

## 작업 내역

### LimitOverrideStore — 저장 추상화

**책임:** 저장-불가지론적 인터페이스. resolver가 Redis 구현이든 인메모리 더블이든 같은 방식으로
fallback을 합성할 수 있게 한다. find는 미설정 = empty, set/clear는 mutation. (인터페이스는
계약이므로 그대로 노출 — 구현은 다음 단위에서.)

```java
public interface LimitOverrideStore {
  Optional<Long> findUserOverride(String userId, LimitType type, Currency currency);
  Optional<Long> findMarketOverride(String marketId, LimitType type, Currency currency);
  void setUserOverride(String userId, LimitType type, Currency currency, long amount);
  void setMarketOverride(String marketId, LimitType type, Currency currency, long amount);
  void clearUserOverride(String userId, LimitType type, Currency currency);
  void clearMarketOverride(String marketId, LimitType type, Currency currency);
}
```

### RedisLimitOverrideStore — subject당 hash 하나

**책임:** 아래 레이아웃을 Redis hash로 구현한다. subject당 hash 하나라 `HGET`이 O(1),
`DEL` 한 번으로 user override 전체 해제. field에 type+currency를 인코딩한 게 그걸 가능케 한다.
저장값은 십진 문자열(counter member 인코딩과 일관), **비-숫자/음수는 fail loud**. 6개 공개
메서드는 prefix만 다른 위임이므로 read/write/field 헬퍼에 로직을 모은다.

```
limit:override:user:{userId}      Redis hash
    field = "{LIMIT_TYPE}:{CURRENCY}"   예: "STAKE_DAILY:KRW"
    value = long minor unit을 십진 문자열로
limit:override:market:{marketId}  같은 모양, 분리된 prefix
```

### 테스트
